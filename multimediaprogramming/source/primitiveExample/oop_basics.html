Object-oriented Programming

In English we refer to a bicycle, a dog, or a car as "things" or "objects." "Object" is a generic term. In programming, we can create objects too. Objects can refer to concrete ideas like a particle or to more abstract ideas like a physics solver. Because the word class refers to a type of objects, we use the terms object and class interchangeably.

class Bicycle{

}

We use objects to create user-defined types.

Fields

  • To store data in objects, we use fields.
  • When an object contains some data we say that the object encapsulates data.
  • Saying that an object encapsulates data is fancy way of saying that an object "has" some data. It might have position information or a colour.

Let's add two fields to our Bicycle class.

class Bicycle{
 int xPos; //store the x-position
 int yPos; //store the y-position
}

Methods

Now that our bike knows where it is, we need to give it the ability to do something. Let's add two methods to our object a method for drawing our bike and one for moving it.

 int yPos; //store the y-position

}

 void moveRight(){

   xPos = xPos + 5;

A Little Theory

When we create an object, we create a reference variable.The reference variable points to the address of the object in memory. When we use the new keyword a reference to the object is created. The new keyword calls the constructor and returns the address of the new object in memory.

This newly create object is called an instance.

Bicycle myBike = new Bicycle();

Variables that are instances of objects are references to the object in memory.
